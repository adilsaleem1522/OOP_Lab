# Object-Oriented Programming (Java)
### COMSATS University Islamabad, Lahore Campus

This repository is a comprehensive collection of my lab work, projects, and exam responses for the 2nd-semester Software Engineering course in Object-Oriented Programming (OOP). It documents my progression from basic syntax to advanced architectural patterns.

## 🛠 Tech Stack
* **Language:** Java
* **Development:**  Notepad, Command Prompt ,and IntelliJ IDEA
* **Version Control:** Git & GitHub

## 📂 Repository Structure & Key Projects


### 🌟 Featured Implementation: Library Membership System (Lab 3)
* **Automated Identity Logic:** Static counters and prefixes to generate unique IDs (e.g., `LIB-0001`).
* **State Management:** Tracking fee status and activity levels for student members.

### 🌟 Featured Implementation: Card Game Engine (Lab 5)
A fully functional card game logic implementation featuring:
* **Custom Deck Management:** 52-card deck with automated shuffling and drawing logic.
* **Complex Mapping:** Use of Enums and custom logic to assign values to Face cards (Ace, King, Queen, Jack).
* **Game Controller:** Multi-player round-based scoring system.

### 🏥 Hospital Management System (Lab 7)
* Advanced use of **Inheritance** and **Abstraction** to model different hospital roles (Doctor, Nurse, Technician).
* Implementation of **Dynamic Binding** and `instanceof` for secure type-casting in staff records.

### 📐 Geometric Shape Suite (Lab 8)
* A polymorphic system for calculating areas and manipulating coordinates of various shapes.
* Uses **Static Members** and **Abstract Base Classes** for shared behavior.

## 🧠 Core OOP Concepts Covered
* **Encapsulation:** Protecting data using private fields and public getters/setters with validation.
* **Inheritance:** Reducing code redundancy by extending base classes.
* **Polymorphism:** Using method overriding and interfaces to treat different objects uniformly.
* **Abstraction:** Defining blueprints through abstract classes and methods.
* **Composition:** Implementing "Has-A" relationships (e.g., a Product has an Expiry Date).
* **Deep Copying:** Using copy constructors to prevent reference-sharing issues.
